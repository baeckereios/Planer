① Kern
Das Herzstück

Zwei Dateien stehen im Zentrum von allem. Alles andere liest sie, schreibt sie, oder baut auf ihnen auf. Ohne sie läuft nichts.

📋
Kern-Datei
stammdaten.js
Die Sollmengen-Bibel der Backstube. Für jedes der 19 Froster-Produkte steht hier drin: wie viel davon an welchem Wochentag gebraucht wird (in Blechen), auf welcher Station es läuft, wie groß eine Charge ist, und ob es einen Grillwetter-Bonus gibt.

Früher wurden diese Werte händisch eingetragen — reines Bauchgefühl und Erfahrung. Heute berechnet der AutoSync aus echten Backmengen-Daten automatisch realistische Durchschnitte und schreibt die Datei neu.
window.BOS_STAMMDATEN = {
  "p1": { name: "Hasenpfoten", needs: [14,14,14,14,16,30,23],
         batchSize: 12, station: "Brötchenstraße", sun: 0 }
}
🗄️
Kern-Datei
backmengen_db.json
Das Gedächtnis der Backstube. Jeden Tag kommt ein neuer Eintrag hinzu — was wurde produziert, wie war das Wetter, waren Ferien, gab es Besonderheiten. Aus dieser wachsenden Datenbank berechnet der AutoSync die Sollmengen für stammdaten.js.

Sie ist das Fundament der datengetriebenen Planung. Je mehr Einträge, desto zuverlässiger werden alle Auswertungen.
{ datum: "2026-03-13", wochentag: "Freitag",
  kontext: { wetter: "sonnig", temperatur: 17, ferien: false },
  produkte: { hasenberger_stueck: 969, ... } }

③ Werkzeuge
Die tägliche Planung

Diese Tools lesen stammdaten.js und machen daraus konkrete Arbeitsanweisungen. Sie sind der eigentliche Nutzen der ganzen Pipeline.

Abhängigkeit: Planer, Froster und Schnellrechner lesen alle window.BOS_STAMMDATEN. Wenn der AutoSync neue Sollwerte berechnet und stammdaten.js aktualisiert wird, rechnen alle drei Tools automatisch mit den neuen Zahlen — ohne Anpassung.
🎯
Planer (PlanungsAssistent)
planer.html + froster_gehirn.js
Das Herzstück der täglichen Vorbereitung. Gibt ein was noch im Froster liegt — das froster_gehirn.js berechnet automatisch was produziert werden muss: Tagesbedarf aus stammdaten.js minus Froster-Bestand = Produktion heute. Berücksichtigt Wochentag, Station, Charge und Grillwetter-Bonus.
🧮
Schnellrechner
schnellrechner.html
Einzelprodukt schnell nachrechnen. Liest Produktliste aus stammdaten.js.
❄️
Frosterliste
frosterliste.html
Druckvorlage A4 für die physische Lagerübersicht. Alle Froster-Produkte auf einem Blatt.
④ Brotrechner
Eigenständig —
kein Stammdaten-Bezug

Das Brot-System läuft komplett unabhängig von stammdaten.js. Es hat seine eigenen Datendateien und seine eigene Logik. Früher war Sauerteig-Berechnung ein täglicher Krampf — das löst dieses Subsystem.

🍞
brot_rechner.html
Zwei Wege: "ich weiß was ich brauche" oder "ich habe fertige Eimer". Berechnet Sauerteig, Vorteig, Zutaten.
🌾
sauerteig_rechner.html
Gezielter Sauerteig- und Vorteig-Rechner für die Nachtschicht.
🧠
brot_daten.js + brot_gehirn.js
Rezepte, Mengenverhältnisse und Berechnungslogik — das Admin-Gegenstück zum Brotrechner.
⑤ BÄKO-Lagersystem
Externe Ware im Griff

Für alles was von BÄKO kommt — Teiglinge, Zutaten, Verpackung. Mittwochs und freitags wird gezählt, der Rest läuft automatisch. Eigenständiges System mit eigener Datenbasis.

Cleverer Trick: Die Bestellung geht per WhatsApp — im Text steckt ein unsichtbarer BOS-SNAP Code mit allen Daten. Du fügst die Nachricht in den Import ein, und die Geschichte wächst automatisch. Mitarbeiter brauchen keinen GitHub-Zugang.
📱
Lager zählen
lager_inventur.html
Mitarbeiter. Bestände eingeben, Lieferung dazu. Feiertag-Haken erhöht Soll automatisch.
📋
Bestellvorschlag
lager_auswertung.html
Was muss bestellt werden? Berechnung automatisch. WhatsApp mit verstecktem BOS-SNAP Code.

Datenbasis: baeko_bestand.js (Stammdaten, Sollmengen) · baeko_historie.js (Verlauf) · lager_gehirn.js (Berechnungslogik)

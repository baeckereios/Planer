🧠 Warum mache ich das hier?

  1. Der Ursprung: Das Team macht Inventur und drückt auf "WhatsApp senden". Dadurch werden Zähldaten erzeugt, die aber noch auf dem Handy des Mitarbeiters gefangen sind.
  2. Der Transfer: Du bekommst die Bestellung per WhatsApp. Unten in der Nachricht hängt ein versteckter System-Code (BOS-SNAP).
  3. Das Füttern: Du kopierst die komplette WhatsApp-Nachricht hier rein und klickst auf Generieren. Dieses Tool "frisst" den Code und klebt ihn in dein Lager-Tagebuch.
  4. Der Server: Du lädst die generierte baeko_historie.js auf deinen Server hoch.
    Warum? Nur so kennt dein Analyse-Tool morgen den Lagerstand von heute und kann den exakten Verbrauch für dich berechnen!
